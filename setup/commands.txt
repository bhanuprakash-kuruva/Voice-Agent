Below is a **fully reproducible beginner-friendly setup guide** based on everything you executed earlier.
This recreates the **same working architecture**:

* **k3d + k3s Kubernetes**
* **Knative Serving**
* **Kourier Ingress**
* **KServe**
* **Ollama runtime**
* **VictoriaMetrics**
* **Grafana**
* **vmagent (Windows metrics collector)**

The steps are intentionally **linear and minimal so a new engineer can copy–paste without debugging**.

---

# 0. Prerequisites (Install First)

Install the following on your machine.

### Install Docker

Download: [https://www.docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop)

Verify:

```bash
docker --version
```

---

### Install kubectl

Windows:

```powershell
winget install Kubernetes.kubectl
```

Verify:

```bash
kubectl version --client
```

---

### Install k3d

```powershell
winget install k3d
```

Verify:

```bash
k3d version
```

---

# TERMINAL 1

# Create Kubernetes Cluster + Install Platform

---

# 1. Check existing clusters

```bash
k3d cluster list
```

If old cluster exists:

```bash
k3d cluster delete kserve-voice-cluster
```

---

# 2. Create the Kubernetes cluster

```bash
k3d cluster create kserve-voice-cluster \
--agents 1 \
--image rancher/k3s:v1.31.5-k3s1 \
-p "11434:30080@agent:0" \
-p "30300:30300@agent:0" \
-p "30750:30750@agent:0" \
-v /usr/share/ollama/.ollama/models:/models@all
```

### What these ports do

| Port  | Service         |
| ----- | --------------- |
| 11434 | Ollama          |
| 30300 | Grafana         |
| 30750 | VictoriaMetrics |

---

# 3. Merge kubeconfig

```bash
k3d kubeconfig merge kserve-voice-cluster --kubeconfig-switch-context
```

Verify context:

```bash
kubectl config current-context
```

---

# 4. Verify cluster

```bash
kubectl get nodes
```

Expected:

```
k3d-kserve-voice-cluster-agent-0
k3d-kserve-voice-cluster-server-0
```

---

# Install Knative Serving

---

# 5. Install Knative CRDs

```bash
kubectl apply -f https://github.com/knative/serving/releases/download/knative-v1.13.1/serving-crds.yaml
```

---

# 6. Install Knative Core

```bash
kubectl apply -f https://github.com/knative/serving/releases/download/knative-v1.13.1/serving-core.yaml
```

Check status:

```bash
kubectl get pods -n knative-serving
```

---

# Install Kourier (Ingress)

---

# 7. Install Kourier

```bash
kubectl apply -f https://github.com/knative/net-kourier/releases/download/knative-v1.13.0/kourier.yaml
```

Verify:

```bash
kubectl get pods -n kourier-system
```

---

# 8. Configure Knative to use Kourier

```bash
kubectl patch configmap/config-network \
-n knative-serving \
--type merge \
-p '{"data":{"ingress-class":"kourier.ingress.networking.knative.dev"}}'
```

Verify:

```bash
kubectl get configmap config-network -n knative-serving -o yaml
```

---

# Install KServe

---

# 10. Install KServe

```bash
kubectl apply -f https://github.com/kserve/kserve/releases/download/v0.12.0/kserve.yaml
```

Verify:

```bash
kubectl get pods -n kserve
```

---

# 11. Fix kube-rbac-proxy image issue

Some environments cannot pull from GCR.

```bash
kubectl set image deployment/kserve-controller-manager \
kube-rbac-proxy=quay.io/brancz/kube-rbac-proxy:v0.13.1 \
-n kserve
```

Restart:

```bash
kubectl rollout restart deployment kserve-controller-manager -n kserve
```

Verify:

```bash
kubectl get pods -n kserve
```

---

# WINDOWS METRICS AGENT (vmagent)

Download:

```
https://github.com/VictoriaMetrics/VictoriaMetrics/releases
```

Download:

```
vmutils-windows-amd64-v1.137.0.zip
```

Extract.

---

# Create vmagent.yml

```yaml
global:
  scrape_interval: 5s

scrape_configs:

  - job_name: voice_rag_pipeline
    metrics_path: /metrics
    static_configs:
      - targets:
        - 127.0.0.1:8000

  - job_name: kube_state_metrics
    metrics_path: /metrics
    static_configs:
      - targets:
        - 127.0.0.1:8080
```

---

# Start vmagent

Open PowerShell inside vmutils folder:

```powershell
.\vmagent-windows-amd64-prod.exe "-promscrape.config=vmagent.yml" "-remoteWrite.url=http://127.0.0.1:30750/api/v1/write"
```
